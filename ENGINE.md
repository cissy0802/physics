## 文风（直觉优先，硬规则）
- **每个概念先给直觉**（类比/可视化/日常场景），公式后跟。物理术语当读者不懂，随写随用大白话解释。
- 松弛、精确、不堆术语吓人；诚实反鸡汤（尤其量子——"意识使波函数坍缩"这类误用要专门辨析，不迎合玄学）。
- 无元叙述：不写"这一期我们讲…""我给你画个图"，正文只留内容。
- 篇幅对齐基准页（Day 1 约 2900 中文字）；`.maxchars`=3600 是硬顶，超了 publish 弹回。

## 公式（硬规则）
- **轻公式一律用自包含 HTML**（如 `.eq`/`.frac` 分数、上下标、`<var>` 斜体变量），**不引 MathJax**——渲染稳、预览不挂。
- **重公式页**（相对论/量子那几期真需要积分、张量、复杂表达式）才用 MathJax，且**自托管进仓库**引本地路径，不热链 CDN。
- **每个符号就地配白话解释**（d 是"微小变化"、每个变量是什么、每个希腊字母读什么）。
- **易混字符加文字标明**：数字 `0`（零）vs 字母 o、`l/1/I` 等，等号右边的 0 放大/标色并写"数字 0"。

## 图（硬规则）
- **一律画 SVG，绝不用等宽 ASCII 图。** 贴合全站暗色调、与基准页协调。
- **必须清晰准确**：布局不挤、文字不重叠出界、箭头/连线渲染后真实可见、对比够。
- **箭头**：斜线上的箭头用 `marker-end` + `orient="auto"`（`markerUnits="userSpaceOnUse"` 定绝对尺寸），别手画三角（朝向写死对不上斜线）；纯横/纯竖的轴对齐箭头可用显式三角。
- **连杆（直线/路径）绝不套 `filter` 辉光或 `objectBoundingBox` 渐变**——轴对齐直线包围盒宽或高为 0，filter/渐变会退化成零导致整条线不渲染只剩箭头浮着。要辉光就底下垫一条无 marker 粗线。
- 能"眼见为实"的就画可演示 SVG。**画完真渲染检查一眼**（headless 截图）：箭头贴线、无浮空三角、无消失连杆。

## 跨期 & 跨站引用（硬规则）
- **跨期引用只用主题名**——不写 Day 编号，也不写位置词（压轴/开篇/上一期/下一期/前面第几期/后面那期）。编号和位置都会因反哺/插入/重排漂移，只有主题名稳定。例：「后面『相变与临界』一期会讲透」。
- **不裸提兄弟站名**（neuroscience/philosophy/meta-knowledge 等）——读者没有全 hub 概念，看到站名很奇怪。跨学科桥梁写成读者能直接读懂的普通概念，或做成真超链接到具体已发布页。

## 每期结构
1. header：kicker（物理 · Physics）+ 渐变/亮色 h1 + `Day N · 年份 · Phase X` + 一句反直觉 tagline
2. `.intro`：why this matters + 一个反直觉转向，无元叙述
3. **3–4 张概念卡**（`.card`），每卡固定 6 段：
   - `.pill` 标签（分支 · 年份/关键词）
   - 【直觉】必须先于公式
   - 【机制】公式(自包含 HTML)+ 每符号解释；需要时插 SVG figure
   - 【反直觉的重点 / 为什么重要】
   - `.xref`【跨学科对读】琥珀框——生物/AI/工程/哲学的**真·紧密**对读；**跨学科别停在口号，每个领域给一个具体例子**（"更多即不同"就给物理/神经/AI/社会各一例）；没有自然对应就不放此框，别硬凑
   - `.punch`【一句话精华 + 思考题】
4. `.deep` 深入思考：3–5 个 `<details>` 开放问题（答案 100–200 字，术语就地解释）
5. `.read` 延伸阅读：**每条可点链接**（Wikipedia / arXiv / 出版社 / 3B1B / Sean Carroll）；只用铁板钉钉的经典，宁缺勿编
